Real-Time Operations Setup

Double-Machine Operations Layout

This project is designed specifically to test Docker deployments across development and operation servers.

1

Development Machine Setup

Your local VS Code workstation is mounted dynamically to the Docker Nginx context. As soon as you modify HTML, CSS, or JS, Nginx serves it instantly!

  • Live Volume Hot Reload: Changes synced instantly.
  • LAN Server Sharing: Machine 2 can access this at http://<dev-ip>:1111.
  • Terminal Access: Run docker compose up -d to serve.
2

Operations Machine Setup

Running a production-ready Nginx Web Server inside Docker, independent of local edits. Gets updated automatically via the automated GitHub Actions CI/CD pipeline.

  • GitHub Webhook Sync: Pushes trigger automatic image rebuilds.
  • Production Nginx Configuration: Optimized cache headers and security controls active.
  • Client Access: Browsable on port 1111 (or mapped port 80).
